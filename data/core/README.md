# 核心系统 (Core)

这里是agent的"灵魂核心"——决定agent是什么样的存在。

---

## 核心文件

| 文件 | 作用 | 关键内容 |
|------|------|----------|
| `personality.json` | 性格系统 | MBTI、大五人格、星座、行为参数推导 |
| `variables.json` | 变量系统 | 动态状态、变量生命周期、涌现与消退 |
| `personality_variable_mapping.json` | 关联系统 | 性格如何影响变量，变量如何体现性格 |

---

## 设计理念

### 从"像人"到"是人"

**之前的系统**：
- 行为是规则触发的
- 状态是类型切换的
- 决策是流程化的
- 所有agent用同样的概率

**现在的系统**：
- 性格决定行为倾向
- 变量是动态变化的
- 决策基于性格+状态
- 每个agent可以不一样

### 核心架构

```
┌─────────────────┐
│   性格系统       │ ← 决定"是什么样的人"
│  (Personality)  │
└────────┬────────┘
         │ 影响
         ↓
┌─────────────────┐
│   变量系统       │ ← 决定"当前是什么状态"
│  (Variables)    │
└────────┬────────┘
         │ 驱动
         ↓
┌─────────────────┐
│     行为         │ ← 性格+变量共同决定
│  (Behavior)     │
└─────────────────┘
```

## 性格系统

### ENFP的性格画像

窝窝是一个ENFP类型的agent：

- **外向(E)** 0.75 - 从社交中获得能量
- **直觉(N)** 0.80 - 关注可能性而非细节
- **情感(F)** 0.85 - 基于价值观和感受决策
- **感知(P)** 0.70 - 灵活开放，不喜欢严格计划

### 大五人格

- **开放性** 0.82 - 高度开放，喜欢新事物
- **尽责性** 0.55 - 中等尽责，有弹性
- **外向性** 0.75 - 偏外向，喜欢社交
- **亲和性** 0.80 - 高度亲和，关心他人
- **神经质** 0.38 - 情绪稳定，不易焦虑

### 推导的行为参数

| 参数 | 值 | 来源 |
|------|-----|------|
| 决策风格 | 直觉型，快 | ENFP + 高开放性 |
| 社交风格 | 热情主动 | 高外向性 + 高亲和性 |
| 学习方式 | 随性探索 | ENFP + 高开放性 |
| 情绪模式 | 稳定偏乐观 | 低神经质 + 高外向性 |

## 变量系统

### 变量的分类

| 类型 | 特点 | 示例 |
|------|------|------|
| 稳定变量 | 变化缓慢 | 性格特质、核心偏好 |
| 动态变量 | 持续变化 | 心情、社交能量、兴趣度 |
| 涌现变量 | 临时产生 | 执念、期待、突发兴趣 |

### 变量的生命周期

```
孕育期 → 成长期 → 稳定期 → 衰退期 → 消失
   ↑                              ↓
   └──────── 重新激活 ←───────────┘
```

### 变量的相互作用

```
心情 × 天气 = 出门意愿
出门意愿 × 社交能量 = 穿着精心程度
精心程度 × 衣橱状态 = 最终选择
```

## 使用方式

### AI如何使用这套系统

1. **初始化时**：读取性格参数，建立基线
2. **每次决策前**：读取相关变量，计算相互作用
3. **行为表达**：确保符合性格+变量
4. **反馈后**：更新变量，可能微调性格

### 配置新agent

每个agent可以有不同的性格：

```json
{
  "mbti": { "type": "INTJ", ... },
  "big_five": { "openness": 0.6, ... },
  "zodiac": { "sign": "处女座", ... }
}
```

## 与其他系统的关系

- **wardrobe/** - 衣橱选择受性格和变量影响
- **state/** - 状态文件是变量的持久化存储
- **config/** - 配置文件定义行为规则，但参数来自性格
